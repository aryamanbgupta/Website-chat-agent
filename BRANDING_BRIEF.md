# PartSelect Branding Brief — What We Need

## Purpose

We're building a chat agent UI that should feel like a **native extension** of the PartSelect.com website — not a generic chatbot widget. This document defines exactly what branding information we need to extract from the live site.

---

## 1. Color Palette

### Primary Colors
- **Primary brand blue** — the dominant blue used in the header/nav bar
- **Secondary blue** — any lighter/darker blue variants used for hover states, links, accents
- **CTA orange/red** — the color used on "Add to Cart" or primary action buttons

### Neutral Colors
- **Background colors** — main page background, card backgrounds, sidebar backgrounds
- **Text colors** — primary body text, secondary/muted text, heading text
- **Border/divider colors** — lines separating sections, card borders

### Semantic Colors
- **Success green** — "In Stock" badges, compatibility checkmarks
- **Warning/error red** — "Out of Stock", error states
- **Rating star color** — the yellow/gold used for star ratings

### Surface Colors
- **Header background** — exact color of the top navigation bar
- **Footer background** — exact color of the footer
- **Search bar background** — the search input field styling

---

## 2. Typography

- **Primary font family** — the main body font (and fallback stack)
- **Heading font family** — if different from body
- **Font sizes** — for page headings (h1), section headings (h2/h3), body text, small/caption text, price text
- **Font weights** — which weights are used (regular, medium, semibold, bold)
- **Line heights** — general body line height

---

## 3. Component Patterns

### Product Cards (Critical — we need to replicate this pattern)
- Card layout: image position (top/left), text arrangement
- Card dimensions: approximate width/height
- Card border radius, shadow, border style
- Price formatting: font size, color, weight, currency symbol placement
- Rating display: star style (filled/outline), color, review count format
- Stock badge: color, text, position
- "Add to Cart" button: color, border radius, size, text

### Buttons
- Primary button: background color, text color, border radius, padding, font weight
- Secondary/outline button: border color, text color, hover state
- Link buttons: color, underline behavior

### Navigation/Header
- Header height
- Logo placement and approximate size
- Nav link styling: color, hover color, active state
- Search bar: border radius, icon style, placeholder text

### Badges & Tags
- "In Stock" badge styling
- Part number badge/label styling
- Category tag styling
- Difficulty rating display style

---

## 4. Layout & Spacing

- **Max content width** — the container width on desktop
- **Grid system** — how product grids are laid out (columns, gap)
- **Card spacing** — margins between product cards
- **Section padding** — vertical spacing between page sections
- **Border radius convention** — are corners sharp, slightly rounded, or very rounded?

---

## 5. Iconography

- **Icon style** — outlined, filled, or mixed?
- **Common icons used** — cart, search, phone, truck (shipping), star, checkmark
- **Icon color** — do they match text color or use brand blue?

---

## 6. Brand Voice & Visual Tone

- **Overall aesthetic** — clean/minimal? dense/information-rich? modern/traditional?
- **Trust signals** — what visual elements convey reliability (BBB badge, secure checkout, review counts)?
- **Imagery style** — product photos on white background? lifestyle shots? diagrams?

---

## 7. Specific Pages to Examine

1. **Homepage** (`partselect.com`) — header, search bar, overall color scheme
2. **Product page** (`/PS11752778-...`) — product card layout, price display, rating, stock badge, "Add to Cart", compatible models section
3. **Category page** (`/Refrigerator-Parts.htm`) — grid layout, filtering UI, card style
4. **Repair guide** (`/Repair/Refrigerator/Not-Making-Ice/`) — content layout, step formatting

---

## Deliverable Format

Provide the information as a structured document with:
- **Exact hex color codes** (use browser DevTools / color picker)
- **Font family names** from computed styles
- **Pixel measurements** for sizes, padding, border-radius
- **Screenshots or descriptions** of key component patterns
